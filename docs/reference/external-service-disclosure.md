# External Service Disclosure

Date: 2026-05-19

This is the canonical WordPress.org disclosure inventory for Flavor Agent. Keep the `readme.txt` external-services block 1:1 with this document whenever outbound call sites, provider routing, background jobs, or data sent to providers changes.

## Submission Summary

Flavor Agent does not send data to external services on plugin activation. Outbound calls are made only after one of these events:

- An administrator configures the related backend in `Settings > Connectors` or `Settings > Flavor Agent`.
- An authorized editor/admin explicitly requests a recommendation, validation, pattern sync, or docs search.
- A background job runs because it was scheduled by one of those configured or user-triggered actions.
- A developer explicitly enables the optional core-roadmap filter.

Browser JavaScript calls only the local WordPress REST API for this site. Third-party calls are server-side through the WordPress HTTP API or delegated to the WordPress AI Client / Connectors runtime.

Recommendation-time Developer Docs grounding is best-effort prompt context. Each recommendation runs at most one cached corpus search; signature-only freshness checks read local caches only and never run live searches. ## Pattern Public-Safety Notes

Cloudflare AI Search for private pattern retrieval uses two public-safety layers:

- Sync-time filtering uploads only public-safe registered patterns and published user `wp_block` patterns across synced, partial, and unsynced states. Private, draft, trashed, or explicitly non-public-safe user patterns are not uploaded.
- Retrieval-time rehydration treats the remote index as stale/untrusted. If a synced pattern's status or permissions change between sync runs, the recommendation request rehydrates the current `wp_block` post and drops candidates that are no longer published and readable before ranking or response output.
