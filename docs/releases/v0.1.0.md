# Flavor Agent v0.1.0 — initial release

Flavor Agent is an AI-assisted recommendation and review layer for WordPress 7.0+ editing surfaces. It brings contextual guidance into the block editor, pattern inserter, Site Editor, Global Styles, Style Book, and wp-admin activity audit flows while keeping mutations bounded, reviewable, and undoable where the plugin owns the apply path.

> Draft these notes into the GitHub release at `https://github.com/henryperkins/flavor-agent/releases/tag/v0.1.0` after tagging `v0.1.0` and attaching the release zip.

## Highlights

- **Native editor coverage:** block Inspector recommendations, post/page content drafting and critique, pattern-inserter recommendations, Site Editor template and template-part recommendations, navigation guidance, Global Styles recommendations, and Style Book recommendations.
- **Review-first safety model:** template, template-part, Global Styles, and Style Book operations are previewed and confirmed before apply; navigation stays advisory-only; content recommendations stay editorial-only; pattern recommendations stay browse/rank-only.
- **Activity and undo:** executable editor actions write server-backed activity records, support scoped undo where the live state still matches, and appear in both inline recent-action sections and `Settings > AI Activity`.
- **WordPress 7.0 Abilities integration:** 20 registered abilities expose recommendation, helper, docs, style, pattern, template, navigation, and infrastructure capabilities with explicit permissions and behavior annotations.
- **Thin REST surface:** remaining plugin REST routes cover activity read/write, activity undo, and admin pattern sync; recommendation execution goes through the WordPress Abilities API.
- **Provider-pluggable architecture:** text generation runs through the WordPress AI Client and `Settings > Connectors`; plugin-owned settings cover embeddings, Qdrant, Cloudflare AI Search pattern retrieval, WordPress docs grounding, guidelines, and pattern sync.

## Included surfaces

| Surface        | v0.1.0 behavior                                                                                                                                 |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| Blocks         | Contextual selected-block recommendations in the Inspector, with safe local attribute apply and guarded structural recommendations.             |
| Content        | Post/page document-panel drafting, editing, and critique suggestions that do not mutate post content automatically.                             |
| Patterns       | Flavor Agent shelf and badge inside the native pattern inserter, matching recommendations against currently allowed/renderable patterns.        |
| Templates      | Site Editor recommendations for `wp_template` documents with review-first bounded operations, activity, stale checks, and undo.                 |
| Template parts | `wp_template_part` recommendations with scoped context, review-first apply for validated operations, activity, and undo.                        |
| Navigation     | Advisory recommendations for selected `core/navigation` blocks; no apply or menu restructuring in this release.                                 |
| Global Styles  | Review-first `theme.json`-safe style operations with activity and undo when live state still matches.                                           |
| Style Book     | Block-example scoped style recommendations using the same review-first executable-surface model.                                                |
| AI Activity    | wp-admin audit page for recent Flavor Agent actions, request/provider metadata, undo state, before/after summaries, and affected-entity links.  |
| Settings       | Connector readiness messaging, embeddings/vector/search configuration, guideline import/export, pattern sync controls, and backend diagnostics. |

## Providers and external services

Flavor Agent does not contact external services on activation and does not own chat credentials. External calls happen after administrator configuration or an explicit authorized user action.

| Capability                      | Supported path                                                                                                                                                           |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Text generation                 | WordPress AI Client via `Settings > Connectors`, including connector-backed providers such as OpenAI or Anthropic when installed/configured.                             |
| Embeddings for Qdrant           | Plugin-owned OpenAI Native or explicitly selected Cloudflare Workers AI settings; previously saved Azure OpenAI embedding options remain legacy-only for older installs. |
| Vector-backed pattern retrieval | Qdrant plus the selected embeddings backend.                                                                                                                             |
| Private pattern search          | Cloudflare AI Search pattern retrieval backend.                                                                                                                          |
| WordPress docs grounding        | Trusted `developer.wordpress.org` chunks through Cloudflare AI Search, cache-only/non-blocking during recommendation flows.                                              |
| Optional core-roadmap guidance  | GitHub public roadmap fetch only when the developer opt-in filter is enabled.                                                                                            |

See [`readme.txt`](../../readme.txt) and [`docs/reference/external-service-disclosure.md`](../reference/external-service-disclosure.md) for the complete disclosure inventory.

## Tested against

- **WordPress:** 7.0 pre-release / trunk-compatible local runtime. The Docker-backed Site Editor harness uses the configured `FLAVOR_AGENT_WP70_BASE_IMAGE` until the official stable WordPress 7.0 image exists.

- **Node/npm:** Node 24.x / npm 11.x default via `.nvmrc`; Node 20.x / npm 10.x also supported by the engine gate.

## Known limitations

- WordPress 7.0 is still pre-release as of this draft; swap the WP 7.0 browser harness to the official stable image once WordPress 7.0 ships.
- Navigation recommendations are advisory-only. There is no validated navigation apply/undo contract in `0.1.0`.
- Pattern recommendations rank and explain native patterns but do not own pattern insertion, pattern undo, or Gutenberg pattern-registry mutation.
- Content recommendations produce editorial guidance and generated text only; there is no automatic post-content patch/apply flow.
- AI Activity is an audit/provenance surface, not a full observability product. Diff-oriented inspection, row-level admin actions, and cross-operator workflows are deferred.
- Global Styles and Style Book suggestions are constrained to validated `theme.json` operations; stronger design-quality or accessibility-quality claims are intentionally deferred until deterministic validation expands.
