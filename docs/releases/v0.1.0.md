# Flavor Agent v0.1.0 — initial release

Flavor Agent is the governance layer for AI-mediated WordPress changes. AI proposes through bounded WordPress contracts; WordPress approves structural/theme changes through native editor and wp-admin review; Flavor Agent records owned applies server-side and only reverses them when drift checks prove undo will not clobber later edits. The recommendation surfaces are the demonstration; the governance layer is the product.

> Draft these notes into the GitHub release at `https://github.com/henryperkins/flavor-agent/releases/tag/v0.1.0` after tagging `v0.1.0` and attaching the release zip.

## Highlights

- **Governed external style applies:** external agents can request bounded Global Styles / Style Book applies through `request-style-apply`; administrators approve or reject them in `Settings > AI Activity`; approved applies execute server-side with freshness and operation revalidation.
- **Review-first safety model:** template, template-part, Global Styles, and Style Book operations are previewed and confirmed before apply; navigation stays advisory-only; content recommendations stay editorial-only; pattern recommendations stay browse/rank-only and are not Flavor Agent-owned apply/undo.
- **Activity, attribution, and drift-safe undo:** executable editor actions and approved external style applies write server-backed activity records, support scoped undo where the live state still matches, and appear in inline recent-action sections or `Settings > AI Activity`.
- **Native proof surfaces:** block Inspector recommendations, post/page content drafting and critique, pattern-inserter recommendations, Site Editor template and template-part recommendations, navigation guidance, Global Styles recommendations, Style Book recommendations, and the wp-admin AI Activity governance console.
- **WordPress 7.0 Abilities integration:** 29 ability contracts expose recommendation, helper/read, docs-search, preview, style, pattern, template, navigation, infrastructure, and external-apply capabilities with explicit permissions and behavior annotations. Helper/read abilities and the five signature-only `preview-recommend-*` siblings register whenever their core contracts are available; recommendation and external-apply abilities also require the WordPress AI feature gate.
- **Thin REST surface:** remaining plugin REST routes cover activity read/write, activity undo, and admin pattern sync; recommendation execution goes through the WordPress Abilities API.
- **Provider-pluggable architecture:** text generation runs through the WordPress AI Client and `Settings > Connectors`; plugin-owned settings cover embeddings, Qdrant, Cloudflare AI Search pattern retrieval, WordPress docs grounding, guidelines, and pattern sync.

| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |

## Providers and external services

Flavor Agent does not contact external services on activation and does not own chat credentials. External calls happen after administrator configuration or an explicit authorized user action.

| Capability                      | Supported path                                                                                                                                                           |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Text generation                 | WordPress AI Client via `Settings > Connectors`, including connector-backed providers such as Anthropic, Google, or OpenAI (the WordPress 7.0 Field Guide's default Connectors screen trio) when installed/configured. |
| Embeddings for Qdrant           | Cloudflare Workers AI settings in `Settings > Flavor Agent`.                                                                                                             |
| Vector-backed pattern retrieval | Qdrant plus Cloudflare Workers AI embeddings.                                                                                                                            |
| Private pattern search          | Cloudflare AI Search pattern retrieval backend.                                                                                                                          |
| WordPress docs grounding        | Trusted `developer.wordpress.org` chunks through the built-in public Cloudflare AI Search endpoint, with cache-first lookup and bounded foreground warming for non-pattern recommendation misses. |
| Optional core-roadmap guidance  | GitHub public roadmap fetch only when the developer opt-in filter is enabled.                                                                                            |

See [`readme.txt`](../../readme.txt) and [`docs/reference/external-service-disclosure.md`](../reference/external-service-disclosure.md) for the complete disclosure inventory.

## Target release configuration

The built-in public Developer Docs corpus was refreshed and validated on 2026-05-19. The `v0.1.0` target release environment enables current-release coverage warnings with:

```php
define( 'FLAVOR_AGENT_DOCS_GROUNDING_REQUIRE_CURRENT_COVERAGE', true );
```

Release environments that cannot define the constant in `wp-config.php` may instead return `true` from the `flavor_agent_docs_grounding_require_current_coverage` filter. Enabling it runs the current Make/Core or Developer Blog coverage probe and surfaces a trusted-but-degraded warning when that source is missing; it does not block recommendations on the release-cycle dimension. Recommendations are still withheld only when no trusted Developer Docs guidance is available (or the coverage probe fails).

Cloudflare AI Search reranking remains disabled for the built-in public Developer Docs endpoint until a dedicated reranking evaluation fixture exists and duplicate/provenance-poor chunks are cleaned up or proven harmless under the trusted-source policy.

## Known limitations

- WordPress 7.0 is still pre-release as of this draft; swap the WP 7.0 browser harness to the official stable image once WordPress 7.0 ships.
- Navigation recommendations are advisory-only. There is no validated navigation apply/undo contract in `0.1.0`.
- External applies are limited to Global Styles and Style Book. Template, template-part, block, content, navigation, and pattern external applies are C2+ work and are not exposed in `0.1.0`.
- Pattern recommendations rank and explain native patterns but do not own pattern insertion, pattern undo, or Gutenberg pattern-registry mutation.
- Content recommendations produce editorial guidance and generated text only; there is no automatic post-content patch/apply flow.
- AI Activity is an approval/audit/provenance surface, not a full observability product. It includes pending external style-apply decisions, structured diff and before/after summaries, but richer visual diff inspection, broader row-level admin actions, and cross-operator workflows are deferred.
- Global Styles and Style Book suggestions are constrained to validated `theme.json` operations; stronger design-quality or accessibility-quality claims are intentionally deferred until deterministic validation expands.
